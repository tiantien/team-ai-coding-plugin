# /code-review - 多维度代码审查

## 功能说明

启动多代理并行代码审查，同时从三个维度深度检查代码质量：
- **复用性**：检查代码重复、可复用性、模块化程度
- **安全性**：检查安全漏洞、敏感信息、权限控制
- **性能**：检查性能瓶颈、资源泄漏、优化空间

## 使用方法

```bash
# 审查当前所有变更
/code-review

# 审查指定文件
/code-review src/services/user-service.ts

# 审查指定目录
/code-review src/api/
```

## 审查流程

### 第一步：收集变更

- 获取 Git 变更文件列表
- 或读取指定文件/目录

### 第二步：并行审查

启动 3 个独立 Agent 分别审查：

#### Agent 1: 复用性审查

检查项目：
- [ ] 是否存在重复代码
- [ ] 函数/组件是否可复用
- [ ] 是否有过度抽象
- [ ] 是否有未使用的代码
- [ ] 模块划分是否合理

#### Agent 2: 安全性审查

检查项目：
- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 风险
- [ ] 是否有敏感信息泄露
- [ ] 权限控制是否完善
- [ ] 输入验证是否充分

#### Agent 3: 性能审查

检查项目：
- [ ] 是否有性能瓶颈
- [ ] 是否有资源泄漏
- [ ] 是否有 N+1 查询
- [ ] 是否有不必要的计算
- [ ] 缓存使用是否合理

### 第三步：汇总报告

生成结构化审查报告：

```markdown
# 代码审查报告

## 概述
- 审查文件数: X
- 发现问题数: Y
- 严重程度: 高/中/低

## 复用性问题
| 文件 | 行号 | 问题 | 建议 |
|------|------|------|------|
| ... | ... | ... | ... |

## 安全性问题
| 文件 | 行号 | 问题 | 严重程度 | 建议 |
|------|------|------|----------|------|
| ... | ... | ... | ... | ... |

## 性能问题
| 文件 | 行号 | 问题 | 影响 | 建议 |
|------|------|------|------|------|
| ... | ... | ... | ... | ... |

## 改进建议
1. ...
2. ...
```

## 问题严重等级

| 等级 | 说明 | 处理方式 |
|------|------|----------|
| 🔴 严重 | 必须立即修复 | 阻塞提交 |
| 🟡 警告 | 建议修复 | 提示用户 |
| 🔵 建议 | 可选优化 | 记录报告 |
