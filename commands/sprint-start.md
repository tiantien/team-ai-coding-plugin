# /sprint-start - 迭代启动初始化

## 功能说明

初始化新的迭代周期，包括：
- 创建迭代分支
- 初始化任务看板
- 设置里程碑
- 配置迭代参数

## 使用方法

```bash
# 启动新迭代
/sprint-start sprint-2026-W17

# 指定迭代目标
/sprint-start sprint-2026-W17 --goal "完成用户中心重构"

# 从需求文档创建迭代
/sprint-start sprint-2026-W17 --from docs/requirements.md
```

## 初始化流程

### 第一步：创建迭代分支

```bash
# 分支命名规范
feature/sprint-{年份}-W{周数}

# 示例
feature/sprint-2026-W17
```

### 第二步：初始化任务看板

创建迭代任务看板文件：

```markdown
# Sprint 2026-W17 任务看板

## 迭代信息
- 迭代名称: Sprint 2026-W17
- 开始日期: 2026-04-21
- 结束日期: 2026-05-04
- 迭代目标: 完成用户中心重构

## 任务列表

### 🔴 进行中
| 任务 | 负责人 | 开始时间 | 预计完成 |
|------|--------|----------|----------|
| - | - | - | - |

### 🟡 待开始
| 任务 | 负责人 | 优先级 | 预估工时 |
|------|--------|--------|----------|
| - | - | - | - |

### ✅ 已完成
| 任务 | 完成人 | 完成时间 | 备注 |
|------|--------|----------|------|
| - | - | - | - |

## 里程碑
- [ ] M1: 设计评审 (2026-04-23)
- [ ] M2: 开发完成 (2026-04-30)
- [ ] M3: 测试完成 (2026-05-03)
- [ ] M4: 发布上线 (2026-05-04)

## 风险项
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| - | - | - |
```

### 第三步：配置迭代参数

创建迭代配置文件 `.claude/sprint-config.json`：

```json
{
  "sprintName": "sprint-2026-W17",
  "startDate": "2026-04-21",
  "endDate": "2026-05-04",
  "goal": "完成用户中心重构",
  "team": [
    {
      "name": "成员1",
      "role": "前端开发"
    },
    {
      "name": "成员2",
      "role": "后端开发"
    }
  ],
  "milestones": [
    {
      "name": "设计评审",
      "date": "2026-04-23"
    },
    {
      "name": "开发完成",
      "date": "2026-04-30"
    },
    {
      "name": "测试完成",
      "date": "2026-05-03"
    },
    {
      "name": "发布上线",
      "date": "2026-05-04"
    }
  ]
}
```

### 第四步：创建迭代文档目录

```
docs/
└── sprints/
    └── sprint-2026-W17/
        ├── README.md           # 迭代概述
        ├── tasks.md            # 任务列表
        ├── design.md           # 设计文档
        ├── meeting-notes.md    # 会议记录
        └── retrospective.md    # 迭代回顾
```

## 从需求文档创建

如果指定了 `--from` 参数，会自动解析需求文档并创建任务：

```bash
/sprint-start sprint-2026-W17 --from docs/requirements.md
```

解析流程：
1. 读取需求文档
2. 提取功能点
3. 生成任务列表
4. 估算工时
5. 分配优先级

## 迭代检查清单

### 迭代开始
- [ ] 创建迭代分支
- [ ] 初始化任务看板
- [ ] 设置里程碑
- [ ] 确认迭代目标
- [ ] 分配任务负责人

### 迭代中
- [ ] 每日更新任务状态
- [ ] 记录阻塞问题
- [ ] 更新风险项

### 迭代结束
- [ ] 完成所有任务
- [ ] 通过测试验收
- [ ] 完成代码审查
- [ ] 合并到主分支
- [ ] 填写迭代回顾
