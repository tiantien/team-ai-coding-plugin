# /deploy - 标准化部署流程

## 功能说明

执行完整的标准化部署流程，包括：
- 部署前检查（测试、构建、安全扫描）
- 部署执行（环境配置、服务部署）
- 部署后验证（健康检查、功能验证）

## 使用方法

```bash
# 部署到开发环境
/deploy dev

# 部署到测试环境
/deploy test

# 部署到预发布环境
/deploy staging

# 部署到生产环境
/deploy prod
```

## 部署流程

### 第一阶段：部署前检查

#### 1. 代码检查
- [ ] 所有测试通过
- [ ] 代码风格检查通过
- [ ] 无 TypeScript/ESLint 错误
- [ ] 无安全漏洞

#### 2. 构建检查
- [ ] 构建成功
- [ ] 无构建警告
- [ ] 资源大小合理
- [ ] 环境变量配置正确

#### 3. 版本检查
- [ ] 版本号已更新
- [ ] CHANGELOG 已更新
- [ ] Git 标签已创建

### 第二阶段：部署执行

#### 开发环境 (dev)
```yaml
流程:
  1. 拉取最新代码
  2. 安装依赖
  3. 执行构建
  4. 部署服务
  5. 重启服务
```

#### 测试环境 (test)
```yaml
流程:
  1. 合并到 test 分支
  2. 执行完整测试套件
  3. 构建生产版本
  4. 部署到测试服务器
  5. 执行数据库迁移
  6. 重启服务
```

#### 预发布环境 (staging)
```yaml
流程:
  1. 创建发布分支
  2. 执行完整测试套件
  3. 执行安全扫描
  4. 构建生产版本
  5. 部署到预发布服务器
  6. 执行数据库迁移
  7. 执行冒烟测试
```

#### 生产环境 (prod)
```yaml
流程:
  1. 确认发布审批
  2. 创建 Git 标签
  3. 执行完整测试套件
  4. 执行安全扫描
  5. 构建生产版本
  6. 执行蓝绿部署
  7. 执行数据库迁移
  8. 执行健康检查
  9. 切换流量
  10. 执行冒烟测试
```

### 第三阶段：部署后验证

#### 健康检查
- [ ] 服务健康检查通过
- [ ] 数据库连接正常
- [ ] 缓存服务正常
- [ ] 外部依赖正常

#### 功能验证
- [ ] 核心功能可用
- [ ] API 响应正常
- [ ] 无错误日志

#### 性能验证
- [ ] 响应时间正常
- [ ] 资源使用正常
- [ ] 无内存泄漏

## 回滚机制

如果部署失败或验证不通过：

```bash
# 自动回滚
1. 切换到上一版本
2. 重启服务
3. 执行健康检查
4. 通知相关人员
```

## 部署检查清单

### 部署前
- [ ] 所有代码已提交
- [ ] 所有测试已通过
- [ ] 版本号已更新

### 部署中
- [ ] 监控部署日志
- [ ] 关注错误告警
- [ ] 准备回滚方案

### 部署后
- [ ] 验证核心功能
- [ ] 检查监控指标
- [ ] 通知相关人员
