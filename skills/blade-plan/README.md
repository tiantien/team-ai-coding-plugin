# Blade Plan

轻量规划执行工作流，Blade Spec 的简化版。将中等复杂度需求快速拆解为结构化任务，逐个执行，全程无辅助文件。

## 核心理念

**先想清楚，再动手写。** 面对涉及多个文件的改动，不急着写代码，而是先理清思路、拆好任务、确认方案，再逐个执行。用最小的流程开销，获得结构化开发的核心收益。

```
需求 → 分析规划 → 确认 → 逐任务执行 → 完成总结
```

## 快速开始

```bash
# 启动规划执行 — 展示方案后等待确认
/blade-plan 给 blade-starter-redis 新增分布式限流注解，支持按IP和用户维度限流

# 自动模式 — 展示方案后直接执行，不等待确认
/blade-plan --auto 重构 OssTemplate，将各厂商实现拆分为独立策略类
```

## 工作流程

**交互模式（默认）：** 展示规划方案后暂停等待确认，审批通过后开始逐任务执行。

```
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  📋 分析与规划                                              │
  │  理解需求 → 扫描代码 → 制定方案 → 拆解任务                 │
  │       ↓ 用户确认                                            │
  │                                                             │
  │  🔨 逐任务执行（TaskCreate/TaskUpdate 追踪进度）            │
  │  Task 1 ✅ → Task 2 ✅ → Task 3 🔄 → Task 4 ⏳ → ...      │
  │       ↓                                                     │
  │  ✅ 完成总结                                                │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

**自动模式（--auto）：** 展示规划后直接开始执行，不等待确认。

```
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  📋 分析规划  →  🔨 逐任务执行  →  ✅ 完成总结             │
  │  （展示方案后直接执行，不暂停）                             │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

## 与 Blade Spec 的定位区分

| 维度 | Blade Plan | Blade Spec |
|------|-----------|------------|
| **适用复杂度** | 中等（5~15 个文件） | 高（15+ 个文件） |
| **流程步骤** | 3 步（分析规划 → 执行 → 总结） | 5 阶段（需求 → 设计 → 任务 → 执行 → 总结） |
| **辅助文件** | 无 | spec.json / requirements.md / design.md / tasks.json / result.md |
| **状态追踪** | Claude Code 内置任务系统 | 文件级状态管理（spec.json + tasks.json） |
| **断点续传** | 不支持（一次会话完成） | 支持跨会话恢复 |
| **审批门控** | 1 次确认 | 每阶段审批 |
| **规划产物** | 对话中展示 | 持久化文档 |

**选择原则**：需求清晰 + 一次会话能完成 → Blade Plan；需求需要分析 + 需要设计文档 + 可能跨会话 → Blade Spec。

## 适用场景

| 场景 | 适合 | 不适合 |
|------|------|--------|
| 新增中等功能模块（如限流注解、审计日志） | ✅ | |
| 跨多个文件的 Bug 修复 | ✅ | |
| 模块级重构（如拆分策略类） | ✅ | |
| 新增 Starter 模块 | ✅ | |
| 现有功能增强（添加新维度、新策略） | ✅ | |
| 单文件简单修改 | | ❌ 直接对话即可 |
| 快速问答或查询 | | ❌ 直接对话即可 |
| 从零开发完整业务系统 | | ❌ 使用 /blade-spec |
| 需要详细架构设计文档 | | ❌ 使用 /blade-spec |

## 特性

- **零文件开销**：不生成任何辅助文件，规划在对话中展示，进度由内置任务系统追踪
- **上下文锚定**：每个任务执行前重读相关代码，防止偏离
- **构建验证**：每组任务完成后自动执行编译检查
- **技能协同**：自动识别可借助 `/blade-design`、`/avue-design` 加速的任务
- **智能升降级**：复杂度超出范围时主动建议切换到 `/blade-spec`，过于简单时直接执行
- **BladeX 体系适配**：内置 Boot/Cloud 分层最佳实践，同时可适配其他技术栈
