# Brainstorm（需求澄清流程）

## 核心目标

通过苏格拉底式问答，把模糊的业务需求拆解为明确的技术规格、边界条件、成功标准，从源头避免需求偏差。

## 触发方式

- 手动触发：`/brainstorm 需求：xxx`

## 标准流程

### 第一步：理解项目上下文

1. 检查项目文件结构
2. 查看相关文档
3. 了解最近提交记录
4. 理解现有架构模式

### 第二步：评估需求范围

- 如果需求描述多个独立子系统，立即标记
- 帮助用户分解为子项目
- 每个子项目独立进行设计流程

### 第三步：逐个提问澄清

- 每次只问一个问题
- 优先使用选择题
- 理解目的、约束、成功标准

### 第四步：提出方案选项

- 提供 2-3 种不同方案
- 说明各方案优缺点
- 给出推荐建议

### 第五步：呈现设计

- 分段呈现设计内容
- 每段确认是否正确
- 涵盖：架构、组件、数据流、错误处理、测试

### 第六步：输出设计文档

- 保存到 `docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md`
- 提交到 Git 仓库

## 提问模板

### 目的澄清

```
这个功能的业务目的是什么？解决什么问题？
```

### 约束确认

```
有哪些技术或业务约束需要考虑？
- 性能要求
- 兼容性要求
- 安全合规要求
```

### 成功标准

```
如何衡量这个功能的成功？
- 功能指标
- 性能指标
- 用户体验指标
```

### 边界确认

```
这个功能的边界在哪里？
- 包含哪些场景
- 不包含哪些场景
```

## 输出物

1. **设计文档**：完整的技术规格说明
2. **成功标准**：可量化的验收指标
3. **风险清单**：潜在风险和应对方案
4. **依赖清单**：外部依赖和前置条件

## 示例

### 输入

```
/brainstorm 需求：给电商系统加一个优惠券核销功能
```

### 输出

```
## 优惠券核销功能设计

### 业务目的
- 支持线下门店核销线上优惠券
- 防止优惠券重复使用

### 功能范围
包含：
- 扫码核销
- 手动输入核销
- 核销记录查询

不包含：
- 优惠券创建（已有系统）
- 优惠券发放（已有系统）

### 成功标准
- 核销响应时间 < 1s
- 支持并发核销 100 QPS
- 核销准确率 100%
- 重复核销拦截率 100%

### 技术方案
[详细设计...]
```
