# Write-Plan（计划拆解流程）

## 核心目标

基于确认的设计文档，把开发任务拆解为 2-5 分钟可完成的最小单元，每个任务明确文件路径、实现逻辑、验证步骤，输出可执行的开发计划。

## 触发方式

- 手动触发：`/write-plan 基于设计文档生成开发计划`

## 标准流程

### 第一步：读取设计文档

- 从 `docs/superpowers/specs/` 读取相关设计文档
- 确认设计文档已通过审核

### 第二步：识别任务单元

1. 按功能模块拆分
2. 按文件拆分
3. 按开发顺序排列

### 第三步：定义任务详情

每个任务包含：
- 任务 ID
- 任务描述
- 目标文件路径
- 实现逻辑
- 验证步骤
- 依赖关系

### 第四步：输出开发计划

保存到 `docs/superpowers/plans/YYYY-MM-DD-<topic>-plan.md`

## 任务拆分原则

### 任务粒度

- 每个任务 2-5 分钟可完成
- 单个任务只涉及一个文件
- 单个任务只有一个明确目标

### 任务依赖

- 明确标注任务间依赖关系
- 无依赖的任务可并行执行
- 有依赖的任务按顺序执行

## 任务模板

```markdown
## 任务 [ID]: [任务名称]

### 描述
[任务描述]

### 目标文件
- [文件路径]

### 实现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

### 验证步骤
1. [验证步骤1]
2. [验证步骤2]

### 依赖
- 前置任务: [任务ID]
- 后置任务: [任务ID]

### 预估时间
[分钟数]
```

## 计划示例

```markdown
# 优惠券核销功能开发计划

## 任务 1: 创建核销服务接口
- 目标文件: src/services/coupon-service.ts
- 实现步骤:
  1. 定义核销请求/响应类型
  2. 创建核销服务类
  3. 实现核销方法签名
- 验证步骤: TypeScript 编译通过
- 依赖: 无
- 预估: 3 分钟

## 任务 2: 实现核销核心逻辑
- 目标文件: src/services/coupon-service.ts
- 实现步骤:
  1. 实现优惠券状态检查
  2. 实现核销记录创建
  3. 实现事务处理
- 验证步骤: 单元测试通过
- 依赖: 任务 1
- 预估: 5 分钟

## 任务 3: 创建核销 API 接口
- 目标文件: src/api/coupon-controller.ts
- 实现步骤:
  1. 创建 POST /coupon/redeem 接口
  2. 添加参数校验
  3. 添加错误处理
- 验证步骤: API 测试通过
- 依赖: 任务 2
- 预估: 4 分钟
```

## 注意事项

- 任务拆分要足够细，避免长会话上下文漂移
- 每个任务要有明确的验证标准
- 标注清楚任务依赖关系
- 预估时间要保守
