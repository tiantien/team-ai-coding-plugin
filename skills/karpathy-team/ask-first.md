# Ask-First Clarification（不确定先询问）

## 核心规则

需求模糊、边界不清晰、存在歧义时，必须先向用户确认，不擅自脑补需求。

## 触发场景

- 用户提交模糊需求时自动触发苏格拉底式追问
- 手动触发：`/clarify-requirements 梳理当前需求的边界和待确认点`

## 使用方法

### 自动触发

当检测到以下情况时，自动触发追问：

- 需求描述缺少关键信息
- 存在多种可能的实现方式
- 边界条件不明确
- 技术选型存在歧义

### 手动触发示例

```
/clarify-requirements 梳理当前需求的边界和待确认点
```

## 需要确认的场景

### 必须确认

1. **功能边界**：需求是否包含某些功能？
2. **技术选型**：使用哪种技术方案？
3. **兼容性要求**：需要支持哪些版本/平台？
4. **性能要求**：有什么性能指标要求？
5. **安全要求**：有什么安全合规要求？

### 可以推断

1. **代码风格**：遵循项目现有风格
2. **目录结构**：遵循项目现有结构
3. **命名规范**：遵循项目现有规范

## 苏格拉底式追问模板

### 模板 1：功能边界确认

```
关于 [功能名称]，我需要确认几个问题：

1. [边界问题1]？
   - 选项 A：xxx
   - 选项 B：xxx

2. [边界问题2]？
   - 选项 A：xxx
   - 选项 B：xxx
```

### 模板 2：技术选型确认

```
关于 [技术选型]，有以下几种方案：

1. 方案 A：xxx（优点：xxx，缺点：xxx）
2. 方案 B：xxx（优点：xxx，缺点：xxx）

请问您倾向于哪种方案？
```

## 示例

### 错误示例

用户：加一个导出功能
AI：（直接实现 Excel 导出）

问题：未确认导出格式、数据范围、权限控制等。

### 正确示例

用户：加一个导出功能
AI：关于导出功能，我需要确认几个问题：

1. 导出格式？
   - Excel
   - CSV
   - PDF

2. 数据范围？
   - 当前页
   - 全部数据
   - 自定义筛选条件

3. 权限控制？
   - 所有人可导出
   - 仅管理员可导出
