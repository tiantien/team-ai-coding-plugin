# Goal-Driven Execution（目标驱动执行）

## 核心规则

只给成功标准，不指定具体实现步骤，让 AI 自主选择最优方案。

## 触发场景

- 用户提交开发需求时自动加载
- 手动触发：`/goal-driven 目标：xxx，成功标准：xxx`

## 使用方法

### 自动触发

当 AI 接收到开发需求时，自动加载此技能，避免过度指令约束。

### 手动触发示例

```
/goal-driven 目标：实现用户登录接口，成功标准：支持JWT鉴权、错误率<0.1%、兼容现有用户体系
```

## 成功标准定义

成功标准必须满足 SMART 原则：

1. **Specific（具体）**：明确描述预期结果
2. **Measurable（可衡量）**：有量化指标
3. **Achievable（可实现）**：在现有条件下可行
4. **Relevant（相关）**：与业务目标相关
5. **Time-bound（有时限）**：有明确的完成时间

## 示例

### 错误示例

```
实现一个登录功能，用 Redis 存储会话，密码用 bcrypt 加密，返回 JSON 格式
```
问题：过度指定实现细节，限制了 AI 的优化空间。

### 正确示例

```
目标：实现用户登录接口
成功标准：
- 支持 JWT 鉴权
- 错误率 < 0.1%
- 兼容现有用户体系
- 响应时间 < 200ms
```

## 注意事项

- 不要指定具体技术栈（除非有明确约束）
- 不要指定具体实现方案
- 专注于"做什么"和"做到什么程度"
- 让 AI 决定"怎么做"
